SECTION 1: OVERVIEW
────────────────────────────────────────────────────────────

This pipeline pulls fulfillment cost data from the ShipBob API for three brands
(4GVN, LiF, GLO) and stores them in Google BigQuery. It is automated via a
Google Cloud Function (GCF) triggered every night at 12:45 AM PST.

Strategy: 3-Day Lookback Sync
- Pulls orders from the last 3 days on each run (not just yesterday).
- Catches late invoice finalization: orders fulfilled today may not have a cost
  until ShipBob's billing runs 24-48 hours later.
- Self-healing: if a nightly run fails, the next run will recover the missed day.
- Deduplication: uses Delete-then-Insert (Upsert) logic to prevent double-counting.

  This fires the Cloud Scheduler immediately, which calls the Cloud Function.

Option B — Via GCP Console (No Terminal):
  1. Go to: https://console.cloud.google.com/cloudscheduler?project=lifbq-407915
  2. Find the job: shipbob-daily-sync-job
  3. Click the three-dot menu (⋮) on the right
  4. Click "Force run"
  5. Confirm. The function will run within 30 seconds.

How to verify it worked:
  Run this query in BigQuery after triggering:

    SELECT
      '4GVN' as brand, COUNT(*) as rows, MAX(ingested_at) as last_sync
    FROM `lifbq-407915.shipbob.4gvn_fulfillment_cost`

    SELECT
      'LIF' as brand, COUNT(*) as rows, MAX(ingested_at) as last_sync
    FROM `lifbq-407915.shipbob.lif_fulfillment_cost`

    SELECT
      'GLO' as brand, COUNT(*) as rows, MAX(ingested_at) as last_sync

    ORDER BY brand

import os
import json
import requests
import time
from google.cloud import bigquery
from datetime import datetime, timedelta

# GCP Configuration
PROJECT_ID = "lifbq-407915"
DATASET_ID = "shipbob"
BRAND_TOKEN_MAP = {
    "4gvn": "SHIPBOB_PAT_4GVN",
    "lif":  "SHIPBOB_PAT_LIF",
    "glo":  "SHIPBOB_PAT_GLO"
}

def shipbob_daily_sync(event, context):
    """
    Cloud Function entry point.
    Triggered by Cloud Scheduler every 24 hours at 12:45 AM PST.
    Syncs last 3 days of ShipBob fulfillment data to BigQuery for all brands.
    """
    client = bigquery.Client(project=PROJECT_ID)

    # 3-Day Lookback to catch late invoice finalization
    lookback_date = (datetime.utcnow() - timedelta(days=3)).strftime('%Y-%m-%dT00:00:00')
    # Note: Earliest available ShipBob data is March 3, 2026 (confirmed via API diagnostic)

    for brand, secret_env_var in BRAND_TOKEN_MAP.items():
        print(f"--- Starting Sync for Brand: {brand.upper()} ---")

        # 1. Get token from environment variable (mapped from Secret Manager)
        token = os.getenv(secret_env_var)
        if not token:
            print(f"[ERROR] Environment variable {secret_env_var} not found. Skipping {brand}.")
            continue

        # 2. API Setup
        url = "https://api.shipbob.com/2026-01/order"
        headers = {
            "Authorization": f"Bearer {token}",
            "Content-Type": "application/json"
        }
        params = {"StartDate": lookback_date, "Limit": 250}

        all_fresh_rows = []
        page = 1

        # 3. Paginated fetch from ShipBob
        while url:
            try:
                print(f"  Fetching {brand} page {page}...")
                resp = requests.get(url, headers=headers, params=params if page == 1 else None, timeout=30)

                if resp.status_code != 200:
                    print(f"  [ERROR] API {resp.status_code}: {resp.text}")
                    break

                orders = resp.json()
                if not orders:
                    break

                for order in orders:
                    shipments = order.get('shipments', [])
                    if not shipments:
                        all_fresh_rows.append({
                            "order_id": str(order['id']),
                            "order_number": str(order.get('order_number') or ''),
                            "status": order.get('status'),
                            "created_date": order.get('created_date'),
                            "shipment_id": None,
                            "shipment_status": None,
                            "invoice_amount": 0.0,
                            "shipped_date": None,
                            "ingested_at": datetime.utcnow().isoformat()
                        })
                    else:
                        for s in shipments:
                            all_fresh_rows.append({
                                "order_id": str(order['id']),
                                "order_number": str(order.get('order_number') or ''),
                                "status": order.get('status'),
                                "created_date": order.get('created_date'),
                                "shipment_id": str(s.get('id')),
                                "shipment_status": s.get('status'),
                                "invoice_amount": float(s.get('invoice_amount') or 0.0),
                                "shipped_date": s.get('shipped_date'),
                                "ingested_at": datetime.utcnow().isoformat()
                            })

                # Check for next page in Link header
                url = None
                link_header = resp.headers.get('Link')
                if link_header:
                    for link in link_header.split(','):
                        if 'rel="next"' in link:
                            url = link.split(';')[0].strip('< >')
                            page += 1
                            break

                time.sleep(0.5)
            except Exception as e:
                print(f"  [EXCEPTION] {e}")
                break

        # 4. Upsert into BigQuery (Delete then Insert)
        if all_fresh_rows:
            table_id = f"{PROJECT_ID}.{DATASET_ID}.{brand}_fulfillment_cost"
            order_ids = list(set([row['order_id'] for row in all_fresh_rows]))

            # Delete existing records for these orders (deduplication)
            delete_query = f"DELETE FROM `{table_id}` WHERE order_id IN UNNEST({json.dumps(order_ids)})"
            client.query(delete_query).result()
            print(f"  Cleared {len(order_ids)} existing records for deduplication.")

            # Insert fresh data
            job_config = bigquery.LoadJobConfig(
                source_format=bigquery.SourceFormat.NEWLINE_DELIMITED_JSON,
                autodetect=True
            )
            client.load_table_from_json(all_fresh_rows, table_id, job_config=job_config).result()
            print(f"  SUCCESS: Ingested {len(all_fresh_rows)} records for {brand}.")
        else:
            print(f"  INFO: No shipments found for {brand} in the lookback window.")

    return "Sync Complete"

SECTION 8: TROUBLESHOOTING
────────────────────────────────────────────────────────────

Issue: invoice_amount is showing $0.00 for some orders
→ Expected. Orders with status "Processing" have not been invoiced yet.
   ShipBob finalizes costs after the carrier scans the package.
   The 3-day lookback window is designed to catch these updates automatically.

Issue: Cloud Function returned a 403 error when triggered manually via browser
→ Expected. The function requires OIDC authentication.
   Use "gcloud scheduler jobs run" or the GCP Console "Force run" button instead.

Issue: A brand shows 0 rows after a sync
→ Check if ShipBob had a 500 error (API instability). This has been observed
   occasionally on the GLO account. The next scheduled run will retry automatically.
   To force a re-sync immediately, use Option A or B in Section 6.

Issue: Duplicate rows in BigQuery
→ Should not happen with the current Upsert logic. If observed, run:
   DELETE FROM `lifbq-407915.shipbob.{brand}_fulfillment_cost`
   WHERE ingested_at < (SELECT MAX(ingested_at) FROM `lifbq-407915.shipbob.{brand}_fulfillment_cost`)
   AND order_id IN (SELECT order_id FROM ... GROUP BY order_id HAVING COUNT(*) > 1)
